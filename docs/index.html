Explore Technology

Core Technology Pillars

Four focused modules power the RoboLab Assistant for moving test tubes (cuvettes) to a DLS analyser.

Vision

A small camera and simple markers tell the robot where the tray, test tube and DLS are. It checks the view before moving in.

Handling

A UR3e robot arm with a gentle gripper lifts fragile test tubes, opens the lid, and can press start if needed.

Path Planning

The software picks a safe, efficient path on a busy bench. Tasks run step by step: open, pick, place, close.

Safety

If something looks wrong, it stops. The goal is around a minute per run, with consistent, accurate placement.

HOW IT WORKS

A Seamless Workflow from Vision to Action

Vision, planning and control work together. Jobs are trialled in simulation first, then run on the robot.

1. Look and confirm

The camera finds the markers and checks the scene matches the plan.

2. Plan the path

The software chooses a safe, efficient move to the next target.

3. Do the action

The robot moves, places the test tube and closes the lid. If misaligned or a spill is seen, it stops.

Project Roadmap

Follow our progress from core functionality to future enhancements.

Completed

As of October 2025

  • Camera can recognise markers and tell us where key items are in 3D.
  • Robot moves are rehearsed in simulation with step-by-step checks.

In Progress

What we're building right now.

  • Make the robot use the camera during motion so it can self-correct in real time.
  • Add an emergency stop and simple spill sensing.

Next (Near Term)

Our immediate goals for the coming months.

  • Plug the camera feed into the motion planner and test on the real robot.
  • Add smart retries if a pick or insert is slightly off, plus the emergency stop path.
  • Prove the robot can open and close the lid gently.
  • Run three full dry cycles in a row with no human help.

Later (Future Directions)

Long-term vision for the project.

  • Reduce reliance on markers with model-based or learned vision.
  • Stress test lighting and angles in simulation before lab time.
  • Ship a lightweight interface to load tray layouts, pick a recipe and monitor runs.
